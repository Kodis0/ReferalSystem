## 1. Цель проекта

- Реализовать **реферальную систему**.
- Разработка ведётся с помощью AI (в Cursor); требования и реализация должны опираться только на то, что вы сообщаете.

## 2. Архитектурное решение (текущее намерение)

- Код **frontend** и **backend** хранится в **одном репозитории** (`frontend/`, `backend/`), при этом слои остаются разделёнными.
- Ранее рассматривался вариант с разными проектами/репозиториями — актуальная структура зафиксирована в корневом `README.md`.

## 4. Ограничения на действия

- **Не начинать реализацию** и **не генерировать код** без отдельной явной команды.
- Сначала — контекст, требования, структура, декомпозиция; реализация — только по запросу.

---

## 5. Текущее состояние репозитория (на момент фиксации)

- **Репозиторий:** ReferalSystem (монорепозиторий).
- **Содержимое:** **frontend** в папке `frontend/` (React, Create React App).
  - Есть: логин, регистрация, домашняя страница, раздел «ЛК» (dashboard, настройки, sidebar), защищённые маршруты, хуки авторизации (`auth.js`, `useCurrentUser.js`).
- **Backend:** в репозитории в папке `backend/` (Django, Django REST Framework, JWT через Simple JWT, CORS; БД по умолчанию SQLite для локального старта, Postgres поддерживается через переменные окружения). Подробности запуска — в корневом `README.md`.
